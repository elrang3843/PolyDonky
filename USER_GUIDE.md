# PolyDonky 사용 방법

PolyDonky 는 HWP · HWPX · DOC · DOCX · HTML · MD · TXT 문서를 한 곳에서 읽고
편집하며 자체 무손실 포맷 **IWPF** 로 보관하는 데스크톱 워드프로세서입니다.
이 문서는 일반 사용자를 대상으로 한 빠른 사용 안내입니다. 자체 포맷의 설계
근거가 궁금하다면 도움말 → "IWPF 포맷" 탭을 함께 참고하세요.

---

## 1. 시작하기

### 새 문서 만들기 / 열기 / 저장

| 동작                       | 메뉴 / 단축키                  |
|---------------------------|-------------------------------|
| 새 문서                    | 파일 → 새 파일 (Ctrl+N)       |
| 파일 열기                  | 파일 → 불러오기 (Ctrl+O)      |
| 저장                       | 파일 → 저장 (Ctrl+S)          |
| 다른 이름으로 저장          | 파일 → 다른 이름으로 저장 (Ctrl+Shift+S) |
| 미리보기 / 인쇄             | 파일 → 미리보기/인쇄 (Ctrl+P) |
| 닫기                       | 파일 → 닫기                   |
| 종료                       | 파일 → 종료 (Alt+F4)          |

* **IWPF · MD · TXT** 는 메인 앱에서 직접 읽고 씁니다.
* **DOCX · HWPX** 는 1급 시민으로 메인 앱에 직접 링크되어 있습니다.
* **HWP · DOC · HTML** 은 별도 CLI 컨버터로 처리되며 (구현 단계에 따라) 외부
  앱이 호출됩니다. 출력 시에는 먼저 IWPF 로 정규화한 뒤 HWPX/DOCX 로
  내보내는 것을 권장합니다.
* 다른 포맷으로 저장할 때는 정보 손실 가능성을 안내하는 확인 다이얼로그가
  한 번 더 뜹니다. **정본은 항상 IWPF 로** 함께 보관하는 것이 안전합니다.

## 2. 본문 편집

### 텍스트 입력과 선택

* 일반 워드프로세서와 동일하게 입력합니다.
* `Insert` 키로 삽입 / 겹쳐쓰기 모드 전환.
* 페이지 끝 빈 공간에서 `Delete` 키를 누르면 다음 페이지의 첫 단락이
  현재 페이지로 끌어올려지고, 페이지 시작 위치에서 `Backspace` 를 누르면
  이전 페이지 끝과 병합됩니다.
* 페이지 경계에서 `↑` / `↓` / `PageUp` / `PageDown` 으로 인접 페이지로
  자연스럽게 캐럿이 이동합니다.

### 글자 / 문단 / 페이지 서식

| 동작            | 단축키 / 위치                         |
|----------------|----------------------------------------|
| 글자 서식        | Ctrl+L / 우클릭 메뉴 / 서식 → 글자 서식 |
| 문단 서식        | Ctrl+T / 우클릭 메뉴 / 서식 → 문단 서식 |
| 페이지 서식      | 서식 → 페이지 서식                     |
| 개요 스타일      | 서식 → 개요 스타일                     |

**글자 서식** — 폰트, 크기, 글자폭(%), 자간(px), 굵게/이탤릭/위첨자/아래
첨자/밑줄/가운데줄/윗줄, 글자색·배경색. 색상 입력칸 옆 스워치 또는 `...`
버튼을 누르면 색상 선택기가 열립니다.

**문단 서식** — 줄 간격, 문단 앞/뒤 여백, 들여쓰기 / 내어쓰기, 정렬
(왼쪽·가운데·오른쪽·양쪽·배분), 자동 번호 / 글머리 기호.

**페이지 서식** — 용지 크기, 방향, 여백, 머리말 / 꼬리말 여백, 단 수,
배경색.

> **⚠️ 방향(가로/세로) 변경 시 단 너비 초기화**
> 다단 모드에서 단 구분선을 드래그해 단 너비를 비대칭으로 조정한 뒤
> 페이지 서식 대화상자에서 방향을 변경하면, 단 너비가 균등 분할로 리셋됩니다
> (각 단의 너비를 같게). 비대칭 단 너비를 유지하려면 방향을 먼저 정한 뒤
> 단 너비를 조정하세요. 저장된 비대칭 단 너비는 파일을 다시 열 때까지
> 그대로 보존됩니다 — 방향을 바꾸지 않는 한.

### 찾기 / 바꾸기

* `Ctrl+F` 또는 `Ctrl+H` 로 찾기/바꾸기 창 열기.
* 대소문자 구분, 단어 단위 일치, 정규식 옵션 제공.

## 3. 객체 삽입

`입력` 메뉴에서 다음을 본문에 삽입할 수 있습니다.

| 객체    | 설명                                                               |
|---------|--------------------------------------------------------------------|
| 글상자   | 사각형, 말풍선, 구름풍선, 가시풍선, 번개상자                          |
| 표      | 엑셀 같은 시트형. 셀 병합·분할, 우클릭 메뉴에서 행/열 추가/삭제·속성 변경 |
| 그래프   | 꺾은선·파이·막대·분포 등 (구현 단계별 활성화)                          |
| 특수문자 | 특수문자 팔레트                                                     |
| 수식    | LaTeX 입력 → 인라인 이미지로 렌더링 (WpfMath)                        |
| 이모지   | 80개 이모지 팔레트 (8개 섹션 × 10개)                                |
| 도형    | 직선, 폴리라인, 폴리곤, 스플라인, 사각형, 원, 호, 화살표 등           |
| 그림    | PNG · BMP · JPEG · TIFF                                             |

도형·이미지·표는 본문 흐름(Inline / Block) 모드와 본문 위(InFrontOfText) /
아래(BehindText) **오버레이 모드** 두 가지 배치를 지원합니다. 객체를 우클릭
하면 배치·속성 메뉴가 나옵니다.

---

## 4. 미리보기 / 인쇄

`파일 → 미리보기/인쇄` 또는 `Ctrl+P` 로 미리보기 창을 엽니다.

* **좌측 패널**: 용지 크기 · 방향 · 여백 · 색상(컬러 / 흑백) · 매수 · 프린터.
* **본문 영역**: 편집창과 완전히 동일한 렌더링 파이프라인으로 그려진 페이지
  미리보기.
* "흑백" 을 선택하면 본문이 즉시 그레이스케일로 표시되고 인쇄 시에도
  PrintTicket 의 OutputColor 가 Monochrome 으로 설정됩니다.
* 좌측 패널에서 프린터를 고르고 `인쇄(P)` 버튼을 누르면 별도의 시스템
  인쇄 다이얼로그 없이 곧바로 선택한 프린터로 전송됩니다.

## 6. 도구

* **설정** — 사용자 정보, 룰러 / 눈금 표시, 언어, 테마 변경, 읽기·쓰기
  포맷 토글.
* **사전** — 외부 사전 모듈 연동 (개발 우선순위 낮음).
* **맞춤법 검사** — 외부 맞춤법 모듈 연동 (개발 우선순위 낮음).
* **사인 만들기** — 별개 독립 앱 (구현 단계별 활성화). 결과 사인은
  `입력` 메뉴의 사인으로 가져와 본문에 삽입할 수 있습니다.

## 7. 외부 포맷과의 호환성

PolyDonky 는 외부 포맷 파일을 **있는 그대로 보관하지 않습니다** — 가져온
순간 IWPF 의 공통 모델로 정규화되며, 동시에 원본의 모든 정보를 살릴 수
있도록 다음 세 층을 함께 보관합니다.

1. **공통 모델** — 편집과 검색이 일어나는 superset 모델.
2. **충실도 캡슐(fidelity capsule)** — DOCX / HWPX / DOC / HWP 등 포맷별
   고유 정보(스타일 시트, 호환성 플래그, OLE 등)를 원형에 가깝게 보관.
3. **원본 파일 내장** — DOC / HWP 처럼 손실 가능성이 큰 포맷은 가져온
   원본 바이트를 패키지 안에 그대로 저장.

따라서 **다른 포맷으로 한 번 내보낸 뒤 다시 PolyDonky 로 불러올 때**,
정본 IWPF 가 함께 있다면 원본에 가깝게 복원할 수 있습니다. 외부 앱이
저장한 결과물은 PolyDonky 의 보존 캡슐을 보존한다는 보장이 없으므로,
PolyDonky 로 다시 가져와 편집할 때는 **IWPF 정본을 함께 들고** 다니는
것이 가장 안전합니다.

자세한 설계 근거는 도움말 → "IWPF 포맷" 탭의 사양 문서를 참고하세요.

## 8. 변경 이력 / 버그 리포트

* 변경 이력은 저장소의 [`HISTORY.md`](HISTORY.md) 에서 확인할 수 있습니다.
* 버그 / 기능 요청은 GitHub Issues 로 제보해 주세요. 가능하면 다음 정보를
  함께 적으면 처리가 빠릅니다.
  * PolyDonky 버전 / 빌드 해시
  * Windows 버전
  * 재현 절차 + 가능한 경우 최소 재현 샘플 (입력 파일)
  * 기대 동작 vs 실제 동작
  * 스크린샷 또는 로그
